# Voice IME 2.0.1 Launch, Test, Problems, and Optimization Backlog

Date: 2026-05-31

2.0.1 is a hardening checkpoint after the 2.0.0 Rust rewrite. It does not claim the product is production-stable yet; it records how to launch it, what has and has not been tested, the current problem points, and the 100 concrete optimizations that should drive the next implementation passes.

## How To Launch

Portable package:

```powershell
Set-Location D:\voice-ime-build-release\voice-ime-2.0.1-rust-portable
& .\启动语音输入.bat
```

Expected result: the main Voice IME GUI appears. Start/stop recording with the large microphone button, or with `Alt+R` if the global shortcut is registered. The cursor overlay appears when recording starts and a target caret can be located; otherwise the main confirmation editor remains the fallback.

Current 2.0.0 backup can still be launched from its own folder:

```powershell
Set-Location D:\voice-ime-build-release\voice-ime-2.0.0-rust-portable
& .\启动语音输入.bat
```

Development mode:

```powershell
Set-Location D:\voice-ime-build-rust
npm install
npm run tauri dev
```

Build and package:

```powershell
Set-Location D:\voice-ime-build-rust
npm run build
npm run tauri build
powershell -ExecutionPolicy Bypass -File .\packaging\package-portable.ps1
```

Packaging must use the Tauri production build. An exe produced only by `cargo build --release` can leave the WebView pointed at `127.0.0.1:1420`, which causes the "refused connection" page when no dev server is running.

## Current Test Truth

Automated tests exist for configuration migration, ASR profile ordering and chunk splitting, audio duration math, correction text rules, edit-command detection, text cleanup, correction trace, and history migration/update. The 2.0.1 regression suite should run:

```powershell
Set-Location D:\voice-ime-build-rust
npm run build
Set-Location src-tauri
cargo check
cargo test
cargo clippy -- -D warnings
Set-Location ..
npm run tauri build
powershell -ExecutionPolicy Bypass -File .\packaging\package-portable.ps1
```

Manual tests not yet completed:

- Real microphone recording across several audio devices.
- Real sherpa-onnx ASR output, because the portable package copied from 1.1.5 currently contains `faster-whisper-small`, not the configured sherpa-onnx ONNX models.
- Settings now exposes clickable download buttons, mirror page buttons, and official model page buttons for missing sherpa-onnx profiles. Downloads try `hf-mirror.com` first, then `huggingface.co`.
- Cursor overlay placement in Notepad, WeChat/Feishu, Chrome inputs, Word/document editors, and IDE input boxes.
- Global hotkey behavior on a clean Windows session and after shortcut conflicts.
- MiniCPM correction/translation with `llama-server` live and with it unavailable.
- Long transcript timing and cancellation under a real 30-120 second recording.
- UI screenshot/DPI pass for main window and overlay.
- Clean-machine portable launch with no existing `.voice_ime` folder.

## Current Problem Points

| Priority | Area | Problem | Impact | Next Check |
| --- | --- | --- | --- | --- |
| P0 partially fixed in 2.0.1 | Models | Default ASR config expects sherpa-onnx model files, but the old package only has `faster-whisper-small`. | Real transcription fails until matching models are downloaded. | Settings now has model download/page/folder actions; full ASR quality still needs real model acceptance. |
| P0 | Integration | No real Windows cross-app input pass has been completed. | Overlay/paste may work in some apps and fail in others. | Test Notepad, WeChat/Feishu, Chrome, Word, IDE. |
| P0 fixed in 2.0.1 | Hotkeys | `AltRight` crashed startup because Tauri/muda did not recognize it. | GUI did not appear at all. | Default changed to `Alt+R`; legacy strings are normalized; hotkey registration failure no longer blocks GUI startup. |
| P0 fixed in 2.0.1 | Packaging | Portable package copied a raw cargo-built exe, so the WebView could open `127.0.0.1:1420` and show refused connection. | Double-clicking the launcher did not show the GUI unless a dev server happened to be running. | Packaging now runs `npm run tauri build`; startup smoke verifies the GUI process stays alive. |
| P0 fixed in 2.0.1 | ASR crash isolation | sherpa-onnx decoding ran inside the GUI process and the personal prompt was passed as hotwords. | Recording completion could close the whole app during ASR setup/decode. | ASR now runs in a subprocess per profile; prompt hotwords are disabled by default; failed profiles fall through to the next profile. |
| P1 | Caret | UIA path uses enclosing element bounding rectangles, not true caret bounding rectangles. | Overlay may appear near a field instead of exactly beside cursor. | Use text range bounding rectangles where available. |
| P1 | LLM | MiniCPM startup is best-effort and silent if model/runtime layout differs. | Smart correction may look "not enabled". | Add service status, logs, and one-click repair. |
| P1 | Packaging | Portable package includes old model folder by copy, which can mislead users. | User expects ASR model to be ready, but it is not. | Replace with explicit model manifest or compatible model pack. |
| P1 | UI | Settings view is functional but too flat; model paths are not editable. | Hard to fix missing models from UI. | Add model path editor and clearer grouped settings. |
| P1 | Diagnostics | No visible log export or environment report. | Debugging "slow/card/not accurate" is guesswork. | Add diagnostics panel and zip export. |
| P2 | State | `Cancelling` exists in frontend type but backend does not expose it. | State machine documentation and UI can drift. | Add real cancelling state or remove it. |
| P2 partially fixed in 2.0.1 | History | History now stores raw ASR, deterministic stages, LLM final text, and stage timings, but has no search/export yet. | Debugging one record is easier; reviewing many records is still clumsy. | Add filters and export. |
